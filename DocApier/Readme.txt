DocApier API服务Readme.txt
DocApier 是负责保存API数据的API服务器，用于帮助AI和用户管理API的编程和开发，本文件既是配置文件，也是使用说明文档。
=============================================
一、基础配置（服务启动核心参数）
Elua = True
Passwd = Memory726
说明：
1. Elua：服务启用开关（True=启动，False=关闭）
2. Passwd：全接口通用鉴权密码，所有API请求必须携带此密码校验
=============================================
二、完整开放API接口定义
1. API搜索接口（根据关键词检索已存储的API）
请求地址：/S
请求方式：GET/POST
请求参数：s=xxx（搜索关键词，如接口名称/路径/功能描述）；pwd=xxx（鉴权密码）
功能：查询已提交存储的API元数据，返回匹配的API列表
适用场景：开发者控制台搜索API、AI工具自动检索可用接口
--------------------------------------------
2. API提交/保存接口（新增/更新API数据）
请求地址：/U
请求方式：POST
请求体（JSON格式）：
{
  "url": "API实际请求地址",
  "home":"API的源服务地址,可以自动向Rust服务器注册（也就是保存到 ../Router/router.router）会追加一条如:From 127.0.0.1:3180 （你的微服务位置）TO /Example（实际上用户可访问的位置）的案例"
  "json": {
    "请求参数结构": "定义入参key/类型/说明",
    "响应结构": "定义返回数据格式",
    "接口描述": "自定义说明",
    "请求方式": "GET/POST/PUT/DELETE"
 },
  "pwd": "鉴权密码"
}
功能：将API信息永久存储到服务数据库，完成API注册
适用场景：AI自动生成API后提交入库、开发者手动注册接口
--------------------------------------------
3. 获取API蓝图接口（获取标准模板/结构定义）
请求地址：/Blueprint
请求方式：GET/POST
请求参数：type=xxx（可选，蓝图类型，如restful/websocket/file/user）；pwd=xxx（鉴权密码）
响应内容：返回标准API开发模板（固定请求格式、响应格式、字段规范）
功能：提供统一的API开发蓝图，保证所有原子API结构标准化
适用场景：AI生成API时拉取标准模板、开发者快速创建规范接口
--------------------------------------------
4. API删除接口（移除已存储的API）
请求地址：/D
请求方式：POST
请求体（JSON格式）：
{
  "url": "需要删除的API地址",
  "pwd": "鉴权密码"
}
功能：根据API路径删除对应元数据
适用场景：接口废弃、版本下线清理
--------------------------------------------
5. API列表查询接口（获取全部已注册API）
请求地址：/List
请求方式：GET
请求参数：pwd=xxx（鉴权密码）
功能：返回服务中所有已存储的API完整列表
适用场景：网关加载路由、控制台展示全部接口
=============================================
三、服务核心功能
1. API持久化存储：永久保存接口地址、请求/响应JSON结构
2. API搜索检索：支持关键词快速查询，方便接口复用
3. 标准化蓝图：统一所有API的开发格式，降低集成成本
4. 权限鉴权：全接口密码校验，保证API数据安全
5. 轻量化无依赖：独立部署、极简配置，可直接对接网关/AI/开发者
=============================================
四、适用场景
1. 配合Rust API网关：提供路由元数据支持
2. 配合AI工具：存储AI生成的API、读取模板自动开发
3. 开发者团队：统一管理内部API资产
4. 原子API架构：作为API元数据中心使用
=============================================
五、使用注意事项
1. 所有API请求必须携带正确的pwd密码，否则会被直接拒绝
2. 提交API时，url为唯一标识，重复提交会覆盖原有API数据
3. 本服务仅存储API元数据，不执行业务逻辑、不转发业务流量
4. 本文件既是服务配置文件，也是完整的使用说明文档，请勿随意修改配置参数
